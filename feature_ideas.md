Feature Specification: Temporal Distribution & Drift Analysis1. Feature OverviewThe Temporal Distribution Analysis module provides continuous monitoring of data streams or batch inferences. It empowers users to visually track the probability distribution of numerical and categorical features over time while simultaneously executing a suite of nonparametric statistical tests to detect structural drift, data degradation, and quality bound violations.2. Scientific & Statistical RationaleStandard mean/variance tracking is insufficient for detecting complex data shifts (e.g., the emergence of bimodality or tail-heaviness). By computing continuous probability density estimates and empirical cumulative distributions across discrete temporal windows, the system can identify subtle structural breaks. The statistical engine relies on nonparametric tests that make no assumptions about the underlying data distribution, ensuring mathematical validity across diverse datasets.3. User Interface Requirements3.1. Control PanelTemporal Grouping (Windowing): Dropdown control allowing users to define the temporal bucket size (e.g., Hourly, Daily, Weekly) or specify dynamic batch IDs.Baseline Definition: Selectors to define the Reference Window (e.g., Training dataset, or a specific past time range like T-7 days) against which all subsequent Current Windows are compared.Visualization Selector: Radio buttons or tabs to switch between plot types: Box Plot, Violin Plot, Histogram, and ECDF.3.2. Visualization PaneMacro View (Continuous): When "Box" or "Violin" is selected, the x-axis represents time (bucketed), and the y-axis represents the feature values. This renders a sequence of plots showing the evolution of the distribution's shape and quantiles over time.Micro View (Comparative): When "ECDF" or "Histogram" is selected, the UI overlays the Current Window's distribution directly on top of the Reference Window's distribution for a single, user-selected time bucket. ECDF is the mathematically preferred default here to eliminate bin-width selection bias inherent in histograms.Visual Alerting (The "Traffic Light" System): Time buckets on the x-axis must be color-coded (Green/Yellow/Red) based on the combined output of the statistical drift engine thresholds.4. Backend & Statistical EngineThe backend will compute the following metrics for every active time bucket against the defined Reference Window.4.1. Nonparametric Hypothesis Testing (p-value based)These tests determine if a statistically significant difference exists between the reference and current distributions.Two-Sample Kolmogorov-Smirnov (KS) Test: * Implementation: Scipy's scipy.stats.ks_2samp.Target: Primarily detects shifts in the central tendency (median) and maximum divergence between empirical distribution functions.Epps-Singleton (ES) Test:Implementation: Scipy's scipy.stats.epps_singleton_2samp.Target: Utilizes the empirical characteristic function. It is highly robust to discrete/binned data and provides superior sensitivity to changes in the spread (variance) and tails of the distribution compared to the KS test.4.2. Distance Metrics (Magnitude based)Because p-values approach zero as sample sizes grow large (even for microscopic variations), distance metrics must be calculated to determine the practical magnitude of the drift.1-Wasserstein Distance (Earth Mover's Distance):Implementation: Scipy's scipy.stats.wasserstein_distance.Target: Calculates the minimum "cost" required to transform the current distribution into the reference distribution. Thresholds should be definable by the user based on feature scale.Population Stability Index (PSI):Implementation: Custom calculation based on a symmetric approximation of the Kullback-Leibler divergence.Target: Measures the shift in proportions across binned data. Standard alerting thresholds apply: PSI < 0.1 (No change), 0.1 <= PSI < 0.2 (Slight change, monitor), PSI >= 0.2 (Significant change, alert).5. Data Quality Layer (Pre-Computation)Before drift metrics are calculated, the pipeline must assess the structural integrity of the temporal bucket. Drift on broken data is a false signal.Completeness Rate: Calculate the percentage of NaN, NULL, or empty string values per bucket.Hard Boundary Constraints: Calculate the percentage of records falling outside physically or logically impossible bounds (e.g., Negative prices, percentages > 100).6. Acceptance CriteriaPerformance: Statistical computations for a window of 100,000 rows must complete in under 5 seconds to ensure UI responsiveness.Mathematical Accuracy: ECDF plots must correctly handle ties in the data without artifacting.Alerting State: A temporal bucket is marked as "Drifting" (Red) only if it violates the Wasserstein or PSI magnitude thresholds, mitigating alert fatigue from hyper-sensitive p-values.Graceful Degradation: If a time bucket contains insufficient data (e.g., $< 5$ samples) to run a mathematically valid ES test, the UI must display a "Low Sample Size" warning rather than throwing an error or generating false positives.
